# UX Audit — ROMI Estimate Calculator
*Senya, 2026-06-12 02:02 PDT | Based on index.html read*

---

## 🔴 Критические баги

### 1. Шаг 3 используется дважды
- Секция "Size (inches)" = step 3
- Секция "Quantity" = тоже step 3
- **Фикс:** Quantity должен быть step 3, Size тоже 3 — но они взаимоисключающие (conditional). Добавить логику: показывать правильный номер шага в зависимости от типа продукта.

### 2. Нет индикатора прогресса
- Пользователь не знает на каком он шаге из скольких
- Карточки появляются снизу по мере заполнения — но нет "breadcrumb" или progress bar
- **Фикс:** Добавить progress dots или числовой счётчик "2 из 5" в header или над текущей секцией

---

## 🟠 UX-проблемы

### 3. Аббревиатуры категорий — непрофессионально
- "Resid." вместо "Residential"
- "Comm." вместо "Commercial"  
- "Special" вместо "Specialty"
- Это технический шорткат, не клиентский язык
- **Фикс:** Убрать сокращения. Если не влезает — увеличить шрифт до 9px или сделать 2 строки

### 4. ZIP-код просят после всего
- Пользователь проходит Category → Type → Size → Options → Advanced → и только потом ZIP
- Но ZIP влияет на цену (zip multiplier). Цена меняется в конце.
- **Фикс:** ZIP поднять выше — или сразу после Category, или в шапку как постоянный инпут

### 5. Advanced Options accordion всегда рендерится
- Блок "5. Advanced Options" показывается даже когда advanced fields пустые (нет options для этого продукта)
- **Фикс:** `if(!advanced.length) return;` уже есть в коде — но display:none при пустых advanced

### 6. "Шаги" не дают feedback что сделано
- После выбора Category нет визуального подтверждения "✓ шаг 1 выполнен"
- Только блок становится активным через `on` класс
- **Фикс:** Добавить галочку или цвет к step-num когда шаг завершён

### 7. Кнопка Admin в хедере
- `/admin` ссылка видна всем — выглядит как dev artifact
- **Фикс:** Убрать из хедера. Доступ к admin через отдельный URL или hidden gesture (5x tap).

### 8. Floor/Technicians/Urgency в секции "Location & Job"
- Urgency (срочность) — это про работу, не про локацию
- Floor (этаж) — это про локацию
- Technicians — это операционный параметр, не про локацию
- Они все смешаны в одну секцию
- **Фикс:** Переименовать секцию → "Job Details" (уже правильно в id `jobSection`) или разбить на 2 карточки

## 🟡 Мелкие улучшения

### 9. Common sizes не подсвечиваются при выборе
- Кнопки 24×36 / 36×48 / 48×60 / 24×24 — не показывают "выбрана" после клика
- **Фикс:** При `setQuickSize` добавлять класс `on` к кнопке, убирать у остальных

### 10. "Calculating..." spinner неинформативен
- Появляется на 400ms при каждом изменении — слишком часто
- **Фикс:** Debounce 600ms + показывать spinner только если запрос > 300ms

### 11. Cart секция — граница слишком яркая
- `border-2 border-green-500` — очень броская на фоне остальных `border-slate-200`
- **Фикс:** `border-green-200` + `bg-green-50` для subtle выделения

### 12. Кнопка "Добавить" в результате — зелёная
- Рядом с синей "Copy" — контраст хороший, но "Добавить" — русский текст среди English UI
- Смешанный язык в одном компоненте
- **Фикс:** Либо весь UI на русский, либо "Add to estimate" вместо "Добавить"

## ✅ Что уже хорошо

- Sticky header — правильно
- `min-height: 44px` у pill кнопок — тач-friendly
- Area preview с live расчётом — отличная фича
- Market badge (РЫНОЧНАЯ/ЗАВЫШЕНА/ЗАНИЖЕНА) — правильная идея
- Smart rounding ($10/$50/$100) — правильно
- Tech recommendation по весу — правильный подход
- Breakdown карточка (material/labor/travel) — нужна, оставить
